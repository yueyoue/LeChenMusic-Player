## 项目概述

**LeChenMusic** 是一个音乐+有声书+影视的 Android APP(Jetpack Compose),服务端基于 Navidrome/Subsonic 协议,影视功能对接 LunaTV。

- **APP端仓库**: https://github.com/yueyoue/LeChenMusic-Player
- **服务端仓库**: https://github.com/yueyoue/LeChenMusic-Server
- **LunaTV 服务**: http://j.tthsdd.top:3000 (admin / shaozi1981)
- **LunaTV 源码**: https://github.com/MoonTechLab/LunaTV
- **参考APP(Selene-Source)**: https://github.com/MoonTechLab/Selene-Source (Flutter,为 LunaTV 配套)
- **Selene-Source 本地克隆**: `/home/work/.openclaw/workspace/Selene-Source` (Flutter/Dart 项目)

APP增加的影视模块全部功能直接参考(Selene-Source)的功能和实现方法,我们遇到问题的时候第一时间先看(Selene-Source)的解决方案引用到我们APP。

## 一、LunaTV 实际 API(已验证)

⚠️ **LunaTV 返回的是裸 JSON,没有 code/msg/data 包装!**

| 接口 | 方法 | 实际返回格式 | 注意事项 |
|------|------|-------------|---------|
| `/api/login` | POST | `{"ok": true}` | body: `{"username":"admin","password":"***"}` |
| `/api/search` | GET | `{"results": [VideoInfo...]}` | **参数用 `q` 不是 `keyword`** |
| `/api/detail` | GET | `VideoDetail` 裸对象 | 参数: `source`, `id` |
| `/api/playrecords` | GET | `{"source+id": {record...}}` | **是 dict 不是 list** |
| `/api/favorites` | GET | `[VideoInfo...]` | 裸数组 |
| `/api/search/resources` | GET | `[SearchResource...]` | 裸数组 |
| `/api/live/sources` | GET | `[LiveSource...]` | - |
| `/api/live/channels` | GET | `[LiveChannelGroup...]` | - |

**不存在的接口**(返回 404):
- ❌ `/api/home`
- ❌ `/api/category`

## 三、未解决的问题(当前代码仍存在)

### 🔴 P0: 点击影片不弹搜索弹窗

**现象**: 点击热门电影/剧集后,不显示"正在搜索播放源"弹窗,直接过很久才跳转(有时跳到播放短剧而非电影)

**分析**: `searchAndPlay` 的 `searchSourceLoading` 状态可能没有正确传递到 HomeScreen 的 LaunchedEffect。搜索完成后 `navigateToDetail` 触发导航,但中间的加载弹窗可能被跳过。

**建议修复方向**:
1. 在 HomeScreen 的 `searchAndPlay` 调用链路加 Log,确认 `searchSourceLoading` 值变化
2. 检查 `navigateToDetail` LaunchedEffect 是否和搜索弹窗的显示逻辑冲突
3. 参考 Selene-Source: 他用一个单独的 `SwitchLoadingOverlay` 组件管理加载状态

### 🔴 P0: 详情页信息区不显示

**现象**: 进入详情页后,封面、标题、评分、年份、类型、导演、演员全部不显示,只有选集

**分析**: `searchAndPlay` 设置了 `videoDetail`,但 `VideoDetailScreen` 的 `LaunchedEffect(source, videoId)` 又调了 `loadDetail`。虽然加了"已有数据跳过"逻辑,但可能 source/id 不匹配导致还是重新加载,而 detail API 返回的数据不完整。

**建议修复方向**:
1. 在 `VideoDetailScreen` 加 Log 打印 `currentDetail` 的值,确认数据是否传入
2. 检查 `loadDetail` 的"已有数据跳过"条件是否正确匹配
3. 如果 detail API 返回的数据不完整,考虑从 `videoDetail` 直接读取(searchAndPlay 已设置完整数据)

### 🔴 P0: 片源切换不显示

**现象**: 详情页没有显示片源选择区域

**分析**: `VideoDetailScreen` 中 `if (sources.size > 1)` 的条件可能不满足。searchAndPlay 只搜索了一个源,`toSources()` 只返回一个源,所以片源选择区域不显示。

**建议修复方向**:
1. searchAndPlay 时搜索多个源(不只是第一个匹配的),存入 `allSources`
2. 详情页显示所有可用源,支持切换
3. 参考 Selene-Source: 他搜索后会保留所有源,并在详情页显示源列表+测速

### 🟡 P1: 内联播放器进度条不能拖动

**现状**: 底部 `LinearProgressIndicator` 只是显示进度,不可交互

**建议**: 改用可拖动的 `Slider` 组件,或使用 Media3 PlayerView 的内置控制器(但需要解决设置按钮遮挡问题)

### 🟡 P1: 全屏切换闪回

**现象**: 点击全屏按钮后立即切回小屏

**分析**: `isPlayerFullscreen` 状态变化触发 LaunchedEffect 改变屏幕方向,但 Compose 重组可能导致状态重置。可能是 `DisposableEffect` 或 `LaunchedEffect` 的 key 问题。

**建议修复方向**:
1. 将 `isPlayerFullscreen` 的 LaunchedEffect key 改为稳定的值
2. 在全屏模式下不释放 ExoPlayer,只是切换 UI
3. 参考 Selene-Source: 他用 `OrientationHelper` 管理屏幕方向,不依赖 Compose 状态

### 🟡 P1: 首页横向导航菜单

**现状**: 菜单已显示(推荐/电影/剧集/动漫/综艺/直播),但点击分类后内容可能为空

**原因**: 分类页用 `searchCategory("电影")` 搜索,但 LunaTV 搜索对泛关键词返回不理想

**建议**: 参考 Selene-Source,分类页用豆瓣 API 获取列表,点击后再搜索 LunaTV 播放

### 🟢 P2: 错误日志上报不工作

**现状**: ErrorReporter 初始化后,服务端未收到错误日志

**分析**: 可能是 `NavidromeAuth.token` 在 ErrorReporter 初始化时还未赋值(用户可能先用影视模块,未登录音乐服务)

**建议**: ErrorReporter 改为支持无认证模式,或在影视登录成功后重新初始化

### 1. 搜索播放流程(核心!)
```
点击电影 → PlayerScreen(title, year)
  → PlayerScreen 内部调 fetchSourcesData(title)
  → 拿到搜索结果(含 episodes)
  → 直接用第一个结果的 episodes 播放
```

- 下方: 片源列表(带测速)+ 选集网格
- 所有内容在一个页面,不需要跳转

## 六、给下位程序员的关键提示 ⚠️

### 1. 不要踩的坑(血泪教训)

| 坑 | 说明 |
|----|------|
| **ExoPlayer HLS** | 必须添加 `media3-exoplayer-hls` 依赖!没有这个依赖播放 m3u8 直接闪退,try-catch 捕获不到(native 层崩溃) |
| **Compose try-catch** | `@Composable` 函数调用不能用 try-catch 包裹,编译器会报错。用 `mutableStateOf` 记录错误状态 |
| **LunaTV 参数** | 搜索用 `q` 不是 `keyword` |
| **LunaTV 返回格式** | 裸 JSON,没有 `code/msg/data` 包装 |
| **播放记录** | 是 dict 不是 list,key 是 `"source+id"` |
| **豆瓣图片** | `img9.doubanio.com` 被拦截,必须替换为 `img3.doubanio.com` |
| **loadDetail 覆盖** | searchAndPlay 已设置 videoDetail 后,VideoDetailScreen 的 LaunchedEffect 会再调 loadDetail 覆盖数据,需要加"已有数据跳过"逻辑 |
| **PlayerView 控件** | Media3 PlayerView 内置控制器有设置按钮,会遮挡自定义按钮。解决方案: `useController = false` + 自定义叠加层 |

### 2. 推荐的修复策略

**当前架构问题**: searchAndPlay → navigateToDetail → VideoDetailScreen 的链路太长,中间状态容易丢失。

**推荐方案**: 参考 Selene-Source,将搜索+详情+播放合并到一个页面:
1. `VideoDetailScreen` 内部自己处理搜索(如果 source 为空)
2. 不依赖 VideoViewModel 的中间状态传递
3. 数据获取和 UI 显示在同一个 Composable 内完成
