# LeChenMusic 影视模块开发总结

## 项目概述

**LeChenMusic** 是一个音乐+有声书+影视的 Android APP(Jetpack Compose),服务端基于 Navidrome/Subsonic 协议,影视功能对接 LunaTV。

- **APP端仓库**: https://github.com/yueyoue/LeChenMusic-Player
- **服务端仓库**: https://github.com/yueyoue/LeChenMusic-Server
- **LunaTV 服务**: http://j.tthsdd.top:3000 (admin / shaozi1981)
- **LunaTV 源码**: https://github.com/MoonTechLab/LunaTV
- **参考APP(Selene-Source)**: https://github.com/MoonTechLab/Selene-Source(Flutter,为 LunaTV 配套)
- APP增加的影视模块全部功能直接参考(Selene-Source)的功能和实现方法,我们遇到问题的时候第一时间先看(Selene-Source)的解决方案引用到我们APP。

## 一、LunaTV 实际 API(已验证)

⚠️ **LunaTV 返回的是裸 JSON,没有 code/msg/data 包装!**

| 接口 | 方法 | 实际返回格式 | 注意事项 |
|------|------|-------------|---------|
| `/api/login` | POST | `{"ok": true}` | body: `{"username":"admin","password":"***"}` |
| `/api/search` | GET | `{"results": [VideoInfo...]}` | **参数用 `q` 不是 `keyword`** |
| `/api/detail` | GET | `VideoDetail` 裸对象 | 参数: `source`, `id` |
| `/api/playrecords` | GET | `{"source+id": {record...}}` | **是 dict 不是 list** |
| `/api/favorites` | GET | `[VideoInfo...]` | 裸数组 |
| `/api/search/resources` | GET | `[SearchResource...]` | 裸数组 |
| `/api/live/sources` | GET | `[LiveSource...]` | - |
| `/api/live/channels` | GET | `[LiveChannelGroup...]` | - |

**不存在的接口**(返回 404):
- ❌ `/api/home`
- ❌ `/api/category`

## 二、已修复的问题(可编译通过)

### 1. API 参数名错误
- **问题**: 搜索接口用了 `keyword` 参数,实际应该是 `q`
- **影响**: 搜索永远返回空结果
- **修复**: `VideoApiClient.kt` 中 `@Query("keyword")` → `@Query("q")`

### 2. DoubanApiClient JVM 签名冲突
- **问题**: `private val api` 属性和 `fun getApi()` 方法在 JVM 层面同名
- **修复**: 属性改名为 `_api`

### 3. 豆瓣 API 返回格式变化
- **问题**: 豆瓣 API 不再返回 `cover_url`,改为 `pic.large`/`pic.normal`
- **修复**: 新增 `DoubanPic` 数据模型,`displayCover` 优先读 `pic.normal`

### 4. 豆瓣图片 418 被拦截
- **问题**: `img9.doubanio.com` 拦截非浏览器请求
- **修复**: CDN 替换为 `img3.doubanio.com`(带 Referer 可访问)
- **参考**: Selene-Source 也用同样方案(`official_cdn` 选项)

### 5. getCategory API 不存在
- **问题**: 调用了不存在的 `/api/category` 接口
- **修复**: 改为用搜索接口 `searchCategory(keyword)` 实现分类

### 6. LunaApiResponse 包装不存在
- **问题**: 代码期望 `code/msg/data` 包装,LunaTV 返回裸 JSON
- **修复**: 重写所有数据模型,移除包装层

### 7. 播放记录格式错误
- **问题**: 代码期望 list,实际返回 dict
- **修复**: 从 dict 解析为 list,key 中提取 source+id

### 8. ✅ 点击热门电影/剧集闪退(2026-07-21 修复)
- **问题**: 点击豆瓣热门电影后,页面空白或闪退
- **根因**: `searchAndPlay` 无加载提示,搜索结果 episodes 为空时直接闪退
- **修复**:
  - `VideoViewModel.searchAndPlay()` 添加 `searchSourceLoading`/`searchSourceMessage` 状态
  - 搜索时显示 AlertDialog 加载弹窗("正在搜索播放源")
  - 优先选择标题匹配 + 有 episodes 的结果
  - episodes 为空时 Toast 提示而不是闪退
  - `HomeScreen` 和 `VideoCategoryScreen` 都添加了加载弹窗
- **参考**: Selene-Source 的 PlayerScreen 自己处理搜索+播放

### 9. ✅ 最近观看点播放闪退(2026-07-21 修复)
- **问题**: 最近观看 → 详情页 → 点播放 → 闪退
- **根因**: 某些源的 detail 返回空 episodes,`toSources()` 返回空列表
- **修复**:
  - `VideoViewModel.loadDetail()` 检测到 episodes 为空时自动调用 `searchAndPlay()` 尝试其他源
  - `VideoDetailScreen` 播放按钮:无资源时显示"搜索播放源"而不是空白
  - `VideoPlayerScreen` 添加空 sources 保护:显示错误提示页而不是闪退
  - `VideoPlayerDirect` 已有空数据保护(确认无遗漏)

## 三、未解决的问题（当前代码仍存在）

### 🟡 问题1: 首页横向导航菜单

**现状**: 菜单已显示（推荐/电影/剧集/动漫/综艺/直播），但点击分类后内容可能为空

**原因**: 分类页用 `searchCategory("电影")` 搜索，但 LunaTV 搜索可能对“电影”这种泛关键词返回不理想

**建议**: 参考 Selene-Source，分类页用豆瓣 API 获取列表，点击后再搜索 LunaTV 播放

### 🟢 问题2: 播放器未嵌入 AndroidView

**现状**: `VideoPlayerScreen` 使用纯 Compose 占位，没有实际嵌入 ExoPlayer 的 `PlayerView`

**影响**: 视频无法真正播放（只有音频，无画面）

**修复方向**: 在播放区域嵌入 `AndroidView(factory = { PlayerView(context).apply { player = exoPlayer } })`

## 五、参考项目 Selene-Source 关键实现

Selene-Source 是 Flutter 项目,但架构思路可参考:

### 1. 搜索播放流程(核心!)
```
点击电影 → PlayerScreen(title, year)
  → PlayerScreen 内部调 fetchSourcesData(title)
  → 拿到搜索结果(含 episodes)
  → 直接用第一个结果的 episodes 播放
```
**不经过详情页,搜索结果直接包含播放链接。**

### 2. 图片处理
```dart
// CDN 替换
originalUrl.replaceAll(RegExp(r'img\d+\.doubanio\.com'), 'img3.doubanio.com')

// 请求头
headers = {
  'Referer': 'https://movie.douban.com/',
  'User-Agent': 'Mozilla/5.0 ...',
  'Accept': 'image/avif,image/webp,...'
}
```

### 3. 搜索参数
```dart
'/api/search', queryParameters: {'q': query.trim()}  // 用 q 不是 keyword
```

### 4. 首页结构
```
TopTabSwitcher (首页/播放历史/收藏夹)
  → ContinueWatchingSection (继续观看)
  → HotMoviesSection (热门电影 - 豆瓣)
  → HotTvSection (热门剧集 - 豆瓣)
  → HotShowSection (热门综艺 - 豆瓣)
  → BangumiSection (动漫 - Bangumi)
```

### 5. 关键文件
- `lib/services/api_service.dart` - API 调用
- `lib/services/douban_service.dart` - 豆瓣数据
- `lib/utils/image_url.dart` - 图片 URL 处理
- `lib/screens/player_screen.dart` - 播放器(搜索+播放一体)
- `lib/screens/home_screen.dart` - 首页

## 六、给下位程序员的建议

### 优先级排序
1. **🔴 P0**: 修复点击播放闪退(最近观看 + 热门电影)
2. **🔴 P0**: 修复热门电影图片不显示
3. **🟡 P1**: 完善播放器功能(投屏、手势、选集)
4. **🟡 P1**: 响应式布局(手机/平板/车机)
5. **🟢 P2**: 分类页内容完善

### 建议的修复策略

**不要**继续在当前架构上打补丁。建议**重写影视播放流程**,参考 Selene-Source:

1. **新建一个 `VideoSearchPlayScreen`**:搜索 + 播放一体化
   - 接收参数: `title`, `year`(来自豆瓣)或 `source`, `id`(来自播放记录)
   - 内部自己搜索 LunaTV、选择源、播放
   - 不依赖 VideoViewModel 的中间状态

2. **简化数据流**:
   - 豆瓣数据 → 只用于首页展示(封面、标题、评分)
   - 点击后 → 搜索 LunaTV → 直接播放
   - 不要走 详情页 → 播放器 的两步流程

3. **图片加载**:
   - 用 `img3.doubanio.com` 替换(已有代码)
   - Coil 拦截器加 Referer(已有代码)
   - 如果还不行,考虑用 LunaTV 服务端代理图片

4. **调试方法**:
   - 在关键方法加 `Log.d("Video", "...")` 打日志
   - 用 `adb logcat | grep Video` 看日志
   - 先确认搜索 API 返回数据正常,再处理 UI

### 需要特别注意的坑
1. LunaTV 搜索用 `q` 不是 `keyword`
2. LunaTV 返回裸 JSON,不要期望 `code/msg/data` 包装
3. 播放记录是 dict 不是 list
4. 豆瓣图片域名会被拦截,必须替换 CDN
5. `VideoInfo` 和 `VideoDetail` 是不同的模型,字段不完全一样
6. 某些源的 detail 返回空 episodes,需要做容错
