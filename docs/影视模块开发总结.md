## 项目概述

**LeChenMusic** 是一个音乐+有声书+影视的 Android APP(Jetpack Compose),服务端基于 Navidrome/Subsonic 协议,影视功能对接 LunaTV。

- **APP端仓库**: https://github.com/yueyoue/LeChenMusic-Player
- **服务端仓库**: https://github.com/yueyoue/LeChenMusic-Server
- **LunaTV 服务**: http://j.tthsdd.top:3000 (admin / shaozi1981)
- **LunaTV 源码**: https://github.com/MoonTechLab/LunaTV
- **参考APP(Selene-Source)**: https://github.com/MoonTechLab/Selene-Source (Flutter,为 LunaTV 配套)

**核心原则**: 所有影视功能直接参考 Selene-Source 的实现方法。遇到问题第一时间看 Selene-Source 的解决方案。

## 一、LunaTV 实际 API(已验证)

⚠️ **LunaTV 返回的是裸 JSON,没有 code/msg/data 包装!**

| 接口 | 方法 | 实际返回格式 | 注意事项 |
|------|------|-------------|---------|
| `/api/login` | POST | `{"ok": true}` | body: `{"username":"admin","password":"***"}` |
| `/api/search` | GET | `{"results": [VideoInfo...]}` | **参数用 `q` 不是 `keyword`** |
| `/api/detail` | GET | `VideoDetail` 裸对象 | 参数: `source`, `id` |
| `/api/playrecords` | GET | `{"source+id": {record...}}` | **是 dict 不是 list** |
| `/api/favorites` | GET | `[VideoInfo...]` | 裸数组 |
| `/api/search/resources` | GET | `[SearchResource...]` | 裸数组 |
| `/api/live/sources` | GET | `[LiveSource...]` | - |
| `/api/live/channels` | GET | `[LiveChannelGroup...]` | - |

**不存在的接口**(返回 404): `/api/home`, `/api/category`

### 豆瓣 API(已验证)

| 接口 | 说明 | 注意事项 |
|------|------|---------|
| `v2/subject/recent_hot/{kind}` | 热门推荐 | `category` 和 `type` 参数,**不支持年份筛选** |
| `v2/{kind}/recommend` | 推荐(带筛选) | `selected_categories` JSON + `tags` 逗号分隔,支持组合筛选 |

**豆瓣 recent_hot 参数映射**(参考 Selene-Source `getCategoryData`):
- 电影: `kind=movie, category=热门/最新/豆瓣高分/冷门佳片, type=全部/华语/欧美/韩国/日本/...`
- 剧集: `kind=tv, category=最近热门, type=tv/美剧/日剧/韩剧/...`
- 动漫: `kind=movie, category=热门, type=日本` (⚠️ 会返回所有日本电影,不精确)
- 综艺: `kind=tv, category=show, type=show`

**豆瓣 recommend 参数映射**(参考 Selene-Source `fetchDoubanRecommends`):
- `selected_categories`: JSON, 如 `{"类型":"喜剧","地区":"韩国"}`
- `tags`: 逗号分隔, 如 `喜剧,韩国,2025`
- `sort`: T(综合)/U(近期热度)/R(首映时间)/S(高分优先)
- `start` + `count`: 分页

```

```

## 六、给下位程序员的关键提示 ⚠️

### 1. 绝对不要踩的坑

| 坑 | 说明 | 后果 |
|----|------|------|
| **ExoPlayer HLS 依赖** | 必须有 `media3-exoplayer-hls` | 没有它播放 m3u8 直接 native 崩溃 |
| **LunaTV 搜索参数** | 用 `q` 不是 `keyword` | 搜索永远返回空 |
| **LunaTV 返回格式** | 裸 JSON,没有 `code/msg/data` | 反序列化失败 |
| **豆瓣 recent_hot 不支持年份** | `category=2025` 返回 0 条 | 年份筛选要用 recommend API |
| **豆瓣动漫 kind=movie** | `kind=movie, type=日本` 返回所有日本电影 | 不是只有动漫 |
| **LaunchedEffect 位置** | 不要放在 LazyColumn `item{}` 内部 | 滚动时被回收,逻辑失效 |
| **ExoPlayer seekTo** | `prepare()` 后不能立即 `seekTo` | 必须等 `STATE_READY` |
| **源切换** | 不要依赖 LaunchedEffect 监听状态变化 | 有延迟和竞争条件,直接在 onClick 中控制 ExoPlayer |
| **详情页方向** | 不要强制 `SCREEN_ORIENTATION_PORTRAIT` | 平板/横屏设备被强制竖屏 |
| **ConcurrentHashMap.contains** | 调用的是 `containsValue` 不是 `containsKey` | 用 `containsKey` 明确调用 |
| **双 trailing lambda** | Kotlin 只允许一个 trailing lambda | 用命名参数 |

### 2. 修改代码的核心原则

**直接控制 ExoPlayer,不依赖中间状态**:
```kotlin
// ✅ 正确
exoPlayer.stop()
exoPlayer.setMediaItem(MediaItem.fromUri(url))
exoPlayer.prepare()
exoPlayer.playWhenReady = true

// ❌ 错误: 通过 ViewModel 状态间接触发(有延迟和竞争条件)
viewModel.switchSource(info)  // → LaunchedEffect → 加载视频
```

**搜索匹配用精确匹配**:
```kotlin
// ✅ 正确
val titleMatch = src.title.replace(" ", "").lowercase() == normalizedTitle
val yearMatch = year.isBlank() || src.year == year
if (titleMatch && yearMatch) { ... }

// ❌ 错误: contains 匹配("晚餐" 会匹配到 "最后的猪肉晚餐")
val matched = sources.firstOrNull { it.title.contains(title, ignoreCase = true) }
```

**筛选用 recommend API,不要用 recent_hot 传年份**:
```kotlin
// ✅ 正确: recommend API + tags
doubanApi.getRecommendations(kind = "movie", tags = "喜剧,2025", ...)

// ❌ 错误: recent_hot 传年份(category=2025 返回 0 条)
doubanApi.getRecentHot(kind = "movie", category = "2025", ...)
```
